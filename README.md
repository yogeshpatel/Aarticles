# Engineering Learning Notes, Labs, and Articles

A personal repository of technical articles, hands-on labs, and architecture notes documenting practical engineering learning across modern software, cloud, and AI-assisted development.

---

## Purpose

This is a practical learning journal — not just a code repository.

It documents real engineering work through working examples, clear explanations, and honest notes on what worked, what failed, and why.

> Build small. Document clearly. Improve continuously.

---

## What's Inside

- Technical articles
- Hands-on lab notes
- Architecture and system design documents
- Backend examples (Java, Python)
- Docker and Kubernetes experiments
- GitOps and Argo CD learning
- CI/CD and DevOps workflows
- Local LLM and RAG experiments
- AI-assisted development notes
- Observability, monitoring, and troubleshooting logs

---

## Who This Is For

- Software and backend engineers
- DevOps and platform engineers
- Engineers learning Docker and Kubernetes
- Developers exploring AI-assisted engineering
- Anyone who prefers practical labs over generic theory

---

## Why Articles Come With Code

Code shows what was built. Articles explain why — the problem, the design decisions, the trade-offs, the failures, and the lessons learned.
